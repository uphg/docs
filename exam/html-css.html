# 1. 必考：你是如何理解HTML语义化的？

HTML语义化就是，用正确的标签做正确的事，例如：标题用 h2 标签，段落用 p 标签，导航用 nav 标签等等

# 2. meta viewport 是做什么的，怎么写？

作用：由于手机端默认会等比例缩放页面，所以需要用 meta viewport 禁止移动端页面缩放，如下：

<!-- VSCode 默认生成 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<!-- 淘宝官网设置 -->
<meta name="viewport" content="width=device-width,initial-scale=1,minimum-scale=1,maximum-scale=1,user-scalable=no,viewport-fit=cover">

# 5. 必考：两种盒模型分别说一下。

概念：CSS 中的 box-sizing 属性定义了 浏览器 应该如何计算一个元素的总宽度和总高度。

box-sizing: border-box;
box-sizing: content-box;

其中

  • border-box 表示当前元素的宽度包括了边框和 padding。
  • content-box 反之，不包括边框和 padding。（默认值）

# 6. 必考：如何垂直居中？

1、table 表格内的元素自带水平垂直居中属性

html

<table>
    <tr>
        <td>内容</td>
    </tr>
</table>

以下CSS均使用如下布局

html

<div class="parent">
    <div class="child"></div>
</div>

2、使用 position 加 margin auto

.parent {
    position: relative;
}
.child {
    position: absolute;
    top: 0;
    bottom: 0;
    left: 0;
    right: 0;
    margin: auto;
}

3、使用 position 加 transform

.parent {
    position: relative;
}
.child {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}

4、使用 flex

.parent {
    display: flex;
    justify-content: center;
    align-items: center;
}

5、使用 vertical-align: middle;

.parent {
    text-align: center;
}
.parent::after, .parent::before {
    content: '';
    height: 100%;
}
.parent::after, .parent::before, .child {
    display: inline-block;
    vertical-align: middle;
}

# 7. 必考：flex 怎么用，常用属性有哪些？

用法：给父元素添加 display: flex; 属性，即可使它的子元素 flex 布局。

常用属性

父元素

display: flex;
/* 项目的排列方向：水平左侧开始，水平右侧开始，垂直上方开始，垂直下方开始。 */
flex-direction: row | row-reverse | column | column-reverse;
/* 换行方式，三个属性：默认、从上到下、从下到上 */
flex-wrap: nowrap | wrap | wrap-reverse;
/* 上面两个属性的简写 */
flex-flow: <flex-direction> || <flex-wrap>;
/* 在主轴上的对齐方式（默认水平方向）：靠左，靠右，居中，两侧无间隙，两侧有间隙 */
justify-content: flex-start | flex-end | center | space-between | space-around;
/* 在交叉轴上如何对齐（默认垂直方向）：上，下，居中，文字底线对齐，沾满高度 */
align-items: flex-start | flex-end | center | baseline | stretch;
align-content: flex-start | flex-end | center | space-between | space-around | stretch;

子元素

/* 项目的排列顺序。数值越小，排列越靠前，默认为0。 */
order: <integer>;
/* 项目的放大比例，默认为 0 */
flex-grow: <number>; /* default 0 */
/* 项目的缩小比例，默认为1，即如果空间不足，该项目将缩小。为 0 时不缩小 */
flex-shrink: <number>; /* default 1 */
/* 可以设为跟width或height属性一样的值（比如350px），则项目将占据固定空间。 */
flex-basis: <length> | auto; /* default auto */
/* flex 属性是 flex-grow, flex-shrink 和 flex-basis的简写，默认值为0 1 auto */
flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]
/* align-self属性允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items属性。
 * 默认值为auto，表示继承父元素的align-items属性，如果没有父元素，则等同于stretch。 */
align-self: auto | flex-start | flex-end | center | baseline | stretch;

# 8. 必考：BFC 是什么？

BFC 全称 Block Formatting Context (块级格式化上下文)

举例回答：我们如果给一个 div 添加一个 overflow: hidden; 那么这个 div 内的浮动元素就会被它包裹起来。这就叫做 BFC。

# 9. CSS 选择器优先级

计算规则

!important > 内联样式 > ID选择器 > 类选择器 > 伪类/属性选择器 > 标签选择器 > 通配符选择器

书写原则

  1. 越具体优先级越高

  2. 写在后面的覆盖写在前面的

  3. !important 权重最高，但尽量少用。

# 10. 列举清除浮动的几种方式

上文中所有带有 BFC 的属性都可以包裹浮动元素，但是都具有副作用，如 overflow: hidden;

推荐写法

clearfix 写在父元素

.clearfix::after {
    content: '';
    display: block;
    clear: both;
}
.clearfix {
    zoom: 1; /* IE兼容 */
}
