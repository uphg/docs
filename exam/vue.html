# 1. 必考：watch 和 computed 和 methods 区别是什么？

computed

  • 计算属性

  • 主要用于数据相加，数据总计等计算功能（一个数据受多个数据影响）

  • 具有缓存性，页面重新渲染时值不变化，计算属性会立即返回之前的计算结果，而不必再次执行函数。

watch

  • 监听数据

  • 主要用于监听当前单个数据的变化（一个数据影响多个数据）

  • 无缓存性，页面重新渲染时值不变化也会执行。

methods

  • methods 主要用于存储常用的一些方法，它需要手动调用才能执行。

# 2. 必考：Vue 有哪些生命周期钩子函数？分别有什么用？

  • beforeCreate 阶段：「在创建组件之前」元素 和 data 都没有被挂载。

  • created 阶段：「在创建组件之后」元素 还没被挂在，但 data 已经被挂载了。

  • beforeMount：「在挂载组件之前」执行时，页面还没有被渲染。

  • mounted：「在挂载组件之后」一般用于请求数据，执行时，页面已经被渲染了。真实DOM已完成，挂载到了页面上，也可以发起Ajax请求。

  • beforeUpdate 「在更新组件之前」

  • updated 「在更新组件之后」

  • beforeDestroy 「在销毁组件之前」

  • destroyed 「在销毁组件之后」

父子组件

var eventBus = new Vue();

# 4. 必考：Vue 数据响应式怎么做到的？

Vue 数据响应式的原理（双向绑定其实 Vue 并没有实现双向绑定）

通过 Object.defineProperty 实现，受 Object.defineProperty 限制，Vue 不能检测 数组和对象的变化。但可以使用 Vue.set(object, propertyName, value)，或vm.$set 实例方法解决

总结：主要通过数据劫持实现，通过 Object.defineProperty 的 get 和 set 实现

上面的 Vue 不能检测 数组和对象的变化：指的是没有在 data 函数中 return 的数据，Vue 是无法检测到的，需要使用 Vue.set(object, propertyName, value) 或 vm.$set

# 6. Vuex 你怎么用的？

  • Vuex 是一个状态管理工具

  • 使用 State 保存数据的值

  • 使用 mutations 处理 state 的数据

  • getters，类似 computed，多个组件公用的计算属性（需要缓存的值）

  • 使用 Action　处理需要异步去请求的数据（就是异步的 mutations）

  • Modules，将上面的四个状态模块化（页面比较多，模块比较多时使用）

# 7. VueRouter 的使用？

以下大部分答案都来自官方文档：https://router.vuejs.org/zh/ (opens new window)

使用

router 路由跳转

// 字符串
router.push('home')
// 对象
router.push({ path: 'home' })
// 命名的路由
router.push({ name: 'user', params: { userId: '123' }})
// 带查询参数，变成 /register?plan=private
router.push({ path: 'register', query: { plan: 'private' }})

组件中使用

<router-link to="/user/detail?id=1">用户1</router-link>

route 获取路由信息

// 获取当前路由
this.$route.path
// 获取路由中传入的参数
this.$route.query.id
// 获取路由中传入的动态参数
this.$route.params

概念

  • Vue Router 是 Vue.js 官方的路由管理器

# 8. Vue 防止样式互相覆盖

可以使用以下三种方式

<!-- 使用 `scoped` attribute -->
<style scoped>
.button { border: none; border-radius: 2px; }
.button-close { background-color: red; }
</style>
<!-- 使用 CSS Modules -->
<style module>
.button { border: none; border-radius: 2px; }
.buttonClose { background-color: red; }
</style>
<!-- 使用 BEM 约定 -->
<style>
.c-Button { border: none; border-radius: 2px; }
.c-Button--close { background-color: red; }
</style>

# 9. Vue Router 中 params 和 query 的区别

区别

  1. query 用 path 来引入，params 用 name 来引入。

  2. query 更加类似于我们 ajax 中 get 传参，params 则类似于 post，说的再简单一点，前者在浏览器地址栏中显示参数，后者则不显示。

共同点

  • 接收参数都是类似的，分别是 this.$route.query.name 和 this.$route.params.name。

关于页面刷新

query 刷新，不会丢失 query 里面的数据（在浏览器地址栏中显示参数）

params 刷新，会丢失 params 里面的数据

# 10. v-if 与 v-for 一起使用

当 v-if 与 v-for 一起使用时，v-for 具有比 v-if 更高的优先级。（不推荐同时使用 v-if 和 v-for）
