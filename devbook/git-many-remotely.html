在一台电脑上同时使用Gitee（码云）和Github

注：此教程的两个账号邮箱需一至

# 创建 ssh key

# 进入用户目录下的 .ssh 文件夹下，路径会因你使用的操作系统不同而略有差异
# 没有这个文件夹也无所谓，直接运行下一句命令也可以
cd ~/.ssh

# 生成 key，将邮件地址替换为你 Gitee 或者 Github 使用的邮件地址
ssh-keygen -t rsa -C "xxx@xxx.com"

# 创建配置文件

在 .ssh 文件夹中创建 config (没有后缀)文件，添加以下内容以区分两个远程仓库

# gitee
Host gitee.com
HostName gitee.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa

# github
Host github.com
HostName github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa

上述配置文件中使用到的配置字段信息简单概述：

  • Host 它涵盖了下面一个段的配置，我们可以通过他来替代将要连接的服务器地址。 这里可以使用任意字段或通配符。 当ssh的时候如果服务器地址能匹配上这里Host指定的值，则Host下面指定的HostName将被作为最终的服务器地址使用，并且将使用该Host字段下面配置的所有自定义配置来覆盖默认的/etc/ssh/ssh_config配置信息。
  • Port 自定义的端口。默认为22，可不配置
  • User 自定义的用户名，默认为git，可不配置
  • HostName 真正连接的服务器地址
  • PreferredAuthentications 指定优先使用哪种方式验证，支持密码和秘钥验证方式
  • IdentityFile 指定本次连接使用的密钥文件

# 测试连接是否正常

# 测试 GitHub

ssh -T git@github.com

如果显示以下内容，则表示成功

Hi yourname! You've successfully authenticated, but GitHub does not provide shell access.

# 测试 Gitee

ssh -T git@gitee.com

如果显示以下内容，则表示成功

Welcome to Gitee.com, yourname!

# 报错问题

如果出现类似 ssh_dispatch_run_fatal 的报错

找到 C 盘的 hosts 文件，添加报错中的路由地址，格式如下：

xxx.xx.xx.xxx gitee.com

# 将代码同时提交到两个远程仓库

# 关联多个远程仓库

先删除已关联的名为 origin 的远程库

git remote rm origin

关联 GitHub 的远程库

git remote add github git@github.com:xxx/demo.git

再关联 码云 的远程库

git remote add gitee git@gitee.com:xxx/demo.git

注意：重新关联的远程库的名称都被改为了 github 和 gitee

推送到 GitHub，使用命令

git push github master

推送到 码云，使用命令

git push gitee master

第一次推送时如果报错，可使用 -f 强制推送（不推荐在公共项目使用）

git push -f gitee master

如果关联仓库地址错误，可以使用以下命令修改

git remote add xxx git@github.com:aaa.git       # 将本地仓库与远程仓库关联
git remote set-url xxx git@github.com:bbb.git   # 修改关联仓库地址

# 同时推送多个仓库

根目录新建push.sh文件，添加以下内容

if [ -z $1 ];then
parame="update"
else
parame=$1
fi

echo "# 提交代码"
git add .
git commit -m "${parame}"

echo "# 上传至Github:"
git push github master
echo "# 上传至码云"
git push gitee master

添加后运行 sh push.sh 或 sh push.sh "上传xxx项目" 即可快捷推送两个远程仓库

提示

如果要拉取分支更新的信息，也要指定分支名称，如拉取 Github 的更新：

git pull github master

# Git 多远程仓库提交脚本

# 首先配置一个简单的全局脚本

创建一个脚本文件目录 mkdir ~/local

创建一个配置文件，用于配置命令：touch ~/.bashrc

编辑配置文件：start ~/.bashrc

在配置文件中添加以下内容：export PATH="local的绝对路径:$PATH"

通过以下命令得到用户路径：cd ~/local; pwd

将上面的 local的绝对路径 替换为你的用户路径

运行 source ~/.bashrc 使配置生效

假如之前脚本 demo.txt 写在用户目录下，要运行 sh ~/local/demo.txt 才能执行该脚本。

现在你只需要运行 demo.txt 即可。

# 使用脚本自动创建多远程仓库提交脚本文件

运行 cd ~/local

创建 remote.sh 文件，添加以下内容（注：需要将其中的 uphg 替换为你的 GitHub 仓库以及码云仓库的文件名）

pwd

if [ -d $2 ]; then
bale="dist"
else
bale=$2
fi

if [ -d $1 ]; then
echo 'error: need to specify directory'
exit
else
echo "# 第一次关联两个仓库
echo -e \"\n== init ==\"
git init
git add .
git commit -m \"first commit\"
echo -e \"== init end ==\n\"

git remote add github git@github.com:uphg/$1.git
git remote add gitee git@gitee.com:uphg/$1.git

echo \"== push Github ==\"
git push github master:master
echo -e \"== push Github end ==\n\"

echo \"== push Gitee ==\"
git push gitee master:master
echo -e \"== push Gitee end ==\n\"" >> once.sh

echo "if [ -z \$1 ];then
parame=\"update\"
else
parame=\$1
fi

echo -e \"\n== commit code ==\"
git add .
git commit -m \"\${parame}\"
echo -e \"== commit end ==\n\"

echo \"== push Github ==\"
git push github master:master
echo -e \"== push Github end ==\n\"

echo \"== push Gitee ==\"
git push gitee master:master
echo -e \"== push Gitee end ==\n\"" >> push.sh

echo "#!/usr/bin/env sh
set -e
yarn build
cd ${bale}

echo -e \"\n== commit build ==\"
git init
git add -A
git commit -m 'deploy'
echo -e \"== commit end ==\n\"

echo \"== push Github ==\"
git push -f git@github.com:uphg/$1.git master:gh-pages
echo -e \"== push Github end ==\n\"

echo \"== push Gitee ==\"
git push -f git@gitee.com:uphg/$1.git master:gh-pages
echo -e \"== push Gitee end ==\n\"

echo \"== Exit the build folder ==\"
cd -" >> deploy.sh
exit
fi

命令语法

remote.sh [要提交的仓库目录] [要部署的文件名(不填默认dist)]

创建脚本后的功能（只在当前目录下生效）

  • 执行 once.sh 即可创建与远程仓库的链接。
  • 执行 push.sh "update" 自动向远程仓库提交。
  • 执行 deploy.sh 将文件打包，部署至 gh-pages 分支。
